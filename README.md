# Atlas Schema Management Starter

This project is a clean starter for managing a database schema with [Atlas](https://atlasgo.io/).

## Project layout

- `atlas.hcl` - Atlas project configuration
- `schema.hcl` - Desired database schema (source of truth)
- `migrations/` - Versioned SQL migrations generated by Atlas
- `AGENTS.md` - Instructions for AI assistants working with Atlas

## Prerequisites

1. Install Atlas CLI:
   - macOS (Homebrew): `brew install ariga/tap/atlas`
2. Install Docker Desktop (used for Atlas dev database URL).
3. Ensure your target database is running (default below assumes local PostgreSQL).

## Quick start

1. Inspect schema source in `schema.hcl`.
2. (Optional) Override database URL:
   - `export ATLAS_DB_URL='postgres://postgres:postgres@localhost:5432/app?sslmode=disable'`
3. Generate a migration from schema changes:
   - `atlas migrate diff init --env local`
4. Validate migration quality:
   - `atlas migrate lint --env local --latest 1`
5. Dry-run apply:
   - `atlas migrate apply --env local --dry-run`
6. Apply for real:
   - `atlas migrate apply --env local`

## Common workflow

1. Edit `schema.hcl`.
2. Run `atlas migrate diff <name> --env local`.
3. Run `atlas migrate lint --env local --latest 1`.
4. Run `atlas migrate apply --env local --dry-run`.
5. Run `atlas migrate apply --env local`.
